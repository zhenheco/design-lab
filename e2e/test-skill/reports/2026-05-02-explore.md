3. 4 個 .js test file 要刪
4. **Plan β6 文案應補上 "改 6 個 .sh inline import"**

**這條沒在 plan 寫 → β6 dispatch Codex 時要在 prompt 補上**

### Risk 2: β5 e2e-smoke break point

`e2e-smoke.test.js:30-49` 流程：`init-library → case-writer 寫 root cases/ → stats.sh 看 Total cases: 1`

β5 改 `stats.js` 用 case-loader.ts (multi-client) 後，stats 改吃 `clients/_personal/cases/`，但 `case-writer.js` (β6 才動) 仍寫 root `cases/` → **e2e-smoke `Total cases: 1` 必 fail**

**修法選擇**：
- A: β5 順便改 case-writer.js 也寫 multi-client（範圍擴張，但符合 plan β2 已建 case-writer.ts 的設計）
- B: β5 改 e2e-smoke.test.js 用 case-writer.ts（不從舊 .js import）
- C: β5 維持 stats.js 雙模 fallback（先讀 root cases/ 再讀 _personal）

**建議 β5 dispatch prompt 預先說明這點 + 採 B**（最小擴張，e2e-smoke 已是 v0.2 schema 假設）

### Risk 3: β3 js-yaml dep 衝突

`package.json` 既有 deps：`gray-matter@^4.0.3`（gray-matter 內含 js-yaml@3.x indirect dep）

β3 加直接 dep `js-yaml@^4` 會造成 **dual install**（@3 給 gray-matter / @4 給 case-loader.ts）：
- npm 容忍（兩份共存）
- 增 ~50KB bundle
- 沒 runtime conflict（不同 import path）

**建議 β3 dispatch prompt**：
- pin js-yaml 跟 gray-matter 內版本一致：`js-yaml@^3.14` + `@types/js-yaml@^3`
- 或接受 dual install（簡單但少潔）

### Risk 4: SKILL.md drift（v0.1 文案 → 實際 v0.2 結構）

`SKILL.md` 仍是 v0.1（行 50 寫「`cases/<slug>.md`」、行 60 寫「v0.2 才搬 SQLite」），但實際 vault 結構已是 v0.2 (`clients/_personal/cases/`)。

**Plan ε3 task** 才會 rewrite SKILL.md。但 Phase α 完成後 SKILL.md 跟實作差太多 → user 跑 SKILL.md 指引會走錯路徑（試找 root cases/ 找不到）。

**建議 α5 收尾 commit 時補一條 deprecation note 到 SKILL.md**（小改動），或在 SKILL.md 開頭加 `# WIP — v0.2 in progress, see docs/superpowers/specs/2026-05-02-design-lab-v0.2-sidecar.md`
