**Личная память дизайнерского вкуса для Claude Code, привязанная к брендам.**

Вы смотрите на дизайны, которые вам нравятся, выносите развёрнутые суждения («типографика отличная, но палитра слишком холодная»), и они накапливаются в локальной памяти. Когда вы позже просите Claude Code собрать фронтенд, `/design` загружает этот накопленный вкус, и результат становится больше похож на *вас* — для каждого бренда отдельно, становясь точнее с каждым циклом.

```
   capture                     store                         generate
┌──────────────┐        ┌──────────────────┐         ┌────────────────────┐
│ liked designs│  ───▶  │  Vault (SSOT)     │  ───▶   │ /design <task>     │
│ (URL / image)│        │  cases + aspects  │         │  loads merged      │
│ + verdicts   │        │  style guides     │         │  brand memory →    │
└──────────────┘        │  NEVER rules      │         │  on-taste frontend │
       ▲                └──────────────────┘         └────────────────────┘
       │                         │  ▲
       │                  ┌──────┴──┴───────┐
       └───────────────── │  local sidecar  │  HTTP @ 127.0.0.1:5174
         MCP write tools  │  (Express)      │  + MCP server (7 tools)
                          └─────────────────┘
```

Система **привязана к брендам**: есть базовый собственный бренд (`_personal`), чей вкус перетекает в каждый бренд, плюс именованные бренды, чьи кейсы (cases) никогда не пересекаются друг с другом.

## Как это работает

- **Хранилище (Vault)** — единый источник истины для вашего вкуса, обычная папка с markdown + YAML, которая принадлежит вам и которую вы можете самостоятельно вести в `git` (хранится вне этого репозитория). Содержит **руководства по стилю (style guides)** по каждому бренду, понравившиеся **кейсы (cases)**, **анти-библиотеку (anti-library)** и **правила NEVER (NEVER rules)**.
- **Sidecar** — небольшой постоянно работающий локальный демон на Express (`127.0.0.1:5174`), который отдаёт по HTTP объединённое, ограниченное выборкой (retrieval-scoped) представление хранилища (`/api/context`, `/api/distill/:brand`, …). Эндпоинты чтения открыты для loopback; для записи требуется локальный API-токен.
- **Сервер MCP** — оборачивает sidecar, чтобы агент (например, локальный агент Hermes) мог читать контекст и фиксировать новый вкус с помощью 7 инструментов.
- **`/design`** — потребитель на стороне Claude Code: загружает объединённую память бренда и оформляет её (руководство бренда → следовать, правила NEVER → жёсткие ограничения, понравившиеся кейсы → подражать, анти-кейсы → избегать) в промпт для генерации.

### Ключевые понятия

| Термин | Значение |
|------|---------|
| **Бренд (Brand)** | Область вкуса. `_personal` = собственный бренд (базовый, перетекает во все). Именованные бренды привязаны к конкретным клиентам. |
| **Кейс (Case)** | Один зафиксированный дизайн (скриншот + извлечённые токены) с вашими цитатами и вердиктом. |
| **Аспект (Aspect)** | Вердикт по отдельному измерению кейса — `{dimension, verdict: like\|dislike, note}`. Дизайн редко бывает хорош во всём. |
| **Руководство по стилю (Style guide)** | Markdown-правила по каждому бренду (глобальное руководство собственного бренда + переопределения для отдельных брендов). |
| **Правило NEVER (NEVER rule)** | Жёсткое ограничение с детектором, применяемое линтером к сгенерированному CSS. |
| **Дистилляция (Distillation)** | Кластеризация накопленных сигналов like/dislike в предлагаемых кандидатов на правила NEVER / заметки о стиле — **только с одобрением, никогда не записывается автоматически**. |

## Цикл

- **Вход** — ежедневный cron локального агента Hermes выдаёт кандидатов на дизайн; вы их просматриваете и отвечаете вердиктами по каждому аспекту; агент фиксирует их в хранилище через инструменты MCP.
- **Выход** — в Claude Code: `/design "<task>" <brand> <scenario>` загружает объединённую память бренда и генерирует фронтенд в накопленном вкусе.
- **Накопление** — по мере того как сигналы накапливаются, `distill_taste` кластеризует их в устойчивых кандидатов на правила; вы одобряете; правило попадает в руководство по стилю бренда; `/design` становится точнее.

[`open-design`](https://github.com/zhenheco/open-design) может опционально потреблять тот же контекст в качестве третьей (доступной только для чтения) студии генерации через скилл `design-memory-bridge`.

## Установка

Требования: **Node ≥ 20** (sidecar использует `better-sqlite3` 12.x; Node 26 подходит) и [Claude Code](https://claude.com/claude-code).

Одна команда — клонируйте и запустите установщик:

```bash
git clone https://github.com/zhenheco/design-lab.git
cd design-lab
bash install.sh
```

`install.sh` устанавливает зависимости, собирает дашборд, привязывает скилл к Claude Code (`~/.claude/skills/design-lab`), инициализирует хранилище, запускает sidecar (демон launchd на macOS; автоматически порождается по запросу в остальных случаях) и выводит, как зарегистрировать сервер MCP. Команда идемпотентна — её безопасно запускать повторно после `git pull`.

Проверка:

```bash
curl -s http://127.0.0.1:5174/api/health        # → {"status":"ok"}
```

По умолчанию хранилище находится в `~/Documents/CC Cli/design-library`; переопределите его через `DESIGN_LAB_VAULT_PATH`. Чтобы использовать инструмент снятия скриншотов `capture_url`, также выполните `npx playwright install chromium`.

### Регистрация сервера MCP

Направьте своего агента на stdio-точку входа `skill/mcp/start.sh` — она автоматически обнаружит все 7 инструментов. Например:

```bash
claude mcp add design-lab -- bash "$(pwd)/skill/mcp/start.sh"
```

### Генерация с вашим вкусом

```bash
/design "build a landing hero" <brand> landing
```

## Инструменты MCP

| Инструмент | Назначение |
|------|---------|
| `get_context` | Прочитать ограниченный выборкой контекст бренда (руководство по стилю + кейсы + правила NEVER). |
| `list_clients` | Перечислить бренды. |
| `add_case` | Зафиксировать кейс из локального пути к изображению. |
| `capture_url` | Снять скриншот URL, извлечь актуальные вычисленные дизайн-токены, сохранить как кейс. |
| `add_feedback` | Записать сигнал о вкусе, не привязанный к одному изображению. |
| `edit_style_guide` | Отредактировать глобальное руководство по стилю или руководство отдельного бренда (с защитой от конфликтов по хешу). |
| `distill_taste` | Кластеризовать накопленные сигналы like/dislike в кандидатов на правила (только для чтения; вы одобряете перед сохранением). |

Для записи требуется локальный API-токен (`X-Design-Lab-Token`, считывается для каждого запроса из `~/.claude/state/design-lab/api-token`). Список разрешённых хостов (Host allowlist) + токен защищают sidecar от DNS-rebinding со стороны локальных браузеров.

## Структура проекта

```
skill/
  sidecar/      Express HTTP sidecar (routes + auth middleware)
  mcp/          MCP server wrapping the sidecar (7 tools)
  lib/          case loader/writer, distill aggregator, URL capture, lint

docs/

```

## Дизайнерские решения

См. [`docs/adr/`](docs/adr/):

- **0001** — держать design-lab и open-design как отдельные кодовые базы.
- **0002** — интегрироваться с агентом через сервер MCP, оборачивающий sidecar.
- **0003** — руководства по стилю для каждого бренда, объединяемые в `/api/context`.
- **0004** — аспектная (по отдельным измерениям) обратная связь по кейсам.
- **0005** — дистилляция с одобрением (детерминированные кластеры в sidecar, черновик от LLM + одобрение человеком перед любой записью).

## Безопасность

Локально-ориентированно и под защитой токена. Бесплатный SAST на каждый PR (Semgrep + Gitleaks + Trivy), плюс Dependabot и сканирование секретов. Хранилище и API-токен находятся вне этого репозитория и никогда не коммитятся.
